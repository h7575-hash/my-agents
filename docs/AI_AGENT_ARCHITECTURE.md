# AIエージェント アーキテクチャまとめ

LangGraph で私用AIエージェントを設計する際の参考として、代表的なパターンを整理したドキュメントです。  
出典: [多様な AI エージェント設計パターン22選を比較 #ChatGPT - Qiita](https://qiita.com/syukan3/items/174e43235bde8a1a0694)

---

## 1. 基本パターン（Anthropic Cookbook 5大パターン）

| パターン | 概要 | 向いているタスク |
|---------|------|------------------|
| **Prompt-Chaining** | 複雑なタスクを複数ステップに分割し、前段の結果を次に引き継ぐ | 順序依存が高い処理、ドキュメント分析 |
| **Parallelization** | 依存のないサブタスクを並列実行して高速化 | 同型タスクの一斉処理（大量レビュー要約など） |
| **Routing** | 入力内容で「どのLLM/プロンプトを使うか」を自動切り替え | 多岐にわたる問い合わせ（サポートなど） |
| **Evaluator-Optimizer** | 生成 → 評価 → 修正を繰り返して品質をブラッシュアップ | コード生成、論文作成など高精度必須 |
| **Orchestrator-Workers** | 中央オーケストレーターがタスク分割し、複数ワーカーで並行/段階実行 | 専門性の異なるサブタスクの一元管理 |

## 2. 拡張パターン（6種）

| パターン | 概要 | 向いているタスク |
|---------|------|------------------|
| **ReAct (Reason+Act)** | 思考(Reason)と行動(Act)を交互に実行し、ツール結果(Observation)を反映 | 検索・計算など外部API活用 |
| **Self-Reflective（内省）** | 同一モデルが自分の出力を自己チェック＆修正 | 別評価モデルなしで改善したい場合 |
| **Tree-of-Thought (ToT)** | 複数思考パスを並列に分岐・探索し、評価しながらベスト解を選択 | アイデアブレスト、複雑なパズル |
| **Plan & Execute** | まず全体の計画を立て、それに沿ってステップを実行 | 大きなタスクの工程管理、複数LLM/人員連携 |
| **Human-in-the-Loop** | 要所で人間がレビュー・修正して安全性・品質を担保 | 法的リスクの大きい文書など |
| **RL（強化学習）ベース** | 報酬を設定し、試行錯誤で自律的に最適化 | ゲームAI、継続的マーケティング最適化 |

---

## 3. 発展パターン（6種）

| パターン | 概要 | 向いているタスク |
|---------|------|------------------|
| **Hierarchical / Modular Multi-Agent** | Orchestrator-Workers を多層化し、各レイヤーに専門エージェントを配置 | 大規模プロジェクト、組織的分業 |
| **Emergent Behavior (Multi-Agent)** | 多数エージェントの相互やり取りで創発的な解を狙う | 戦略シミュレーション、ブレスト |
| **Hybrid Symbolic-Neural** | ルールベース/シンボリック推論とLLMを組み合わせる | 医療・法律など厳密性が求められる領域 |
| **Continual Learning** | 運用しながら新情報を取り込み、モデルを常に更新 | ニュース・SNS動向など最新性が重要な分野 |
| **Memory-Centric** | 外部メモリ（ベクトルDB等）で長期的文脈を参照 | ユーザ履歴管理、長期アシスタント |
| **Socratic（対話駆動）** | エージェントがユーザに質問を返しながら最適解を共創 | コンサル、教育のインタラクティブ活用 |

## 4. 未来的パターン（5種）

| パターン | 概要 | 向いているタスク |
|---------|------|------------------|
| **Evolutionary（遺伝的）** | 遺伝的アルゴリズムでエージェントを世代交代しながら進化 | ロボット制御、ゲームAI |
| **Digital Twin / Simulation-Based** | 物理空間・製造ライン等をデジタルツインで再現し仮想環境で試行錯誤 | 工場自動化、交通最適化、自動運転シミュレーション |
| **Neuromorphic / Brain-Inspired** | スパイキングニューラルネット等の脳型ハードを利用 | エッジAI、リアルタイム処理（研究段階） |
| **Self-Aware / Meta-Cognitive** | 自分の知識・認知限界を理解し、不足時は他エージェント/外部にアクセス | 新規領域への挑戦、リスク管理 |
| **Large-Scale Socioeconomic ABM** | 経済主体をエージェント化し社会経済をシミュレーション | 政策評価、都市計画、環境問題 |

## 6. パターン組み合わせ例

### 例1: 精度・安全性重視
- **Prompt-Chaining** → タスクをステップ分割  
- **ReAct** → ツール/API呼び出しと観測のループ  
- **Evaluator-Optimizer** → 最終出力の検証・修正  

### 例2: 多様タスクの一元管理＋長期記憶
- **Routing** → タスク/ドメインに応じてエージェント振り分け  
- **Orchestrator-Workers** → 専門エージェントの一括管理  
- **Memory-Centric** → 履歴・外部DBで長期・横断情報を共有  

### 例3: 創発的アイデア＋人間の担保
- **Tree-of-Thought** → 複数思考パスを探索  
- **Emergent Behavior** → 複数エージェントの議論で創発  
- **Human-in-the-Loop** → 重要ステップで人間がレビュー

## 8. 設計のつながり

- パターンは**組み合わせ可能**で、基本 → 拡張 → 高度 → 未来的と階層的に発展する。  
- 設計の軸は次の3つで整理するとよい:  
  1. **特化した役割をどう協調させるか**  
  2. **評価・フィードバックをどこで入れるか**  
  3. **外部ツール・メモリをどう使うか**  

まずはシンプルなパターンから試し、必要に応じて拡張パターンや組み合わせを検討するのがおすすめです。
